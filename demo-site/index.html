How It Works

Four ideas that make MCP CMS different from everything else.

Block Architecture

Content lives in named HTML blocks. Each block is an independent, editable unit with a clear ID. No tangled templates, no fragile selectors — just clean, addressable content.

<!-- block:hero-headline -->
<h1>Your content here</h1>
<!-- /block:hero-headline -->

MCP Protocol

The Model Context Protocol gives AI assistants structured access to your CMS. Claude, ChatGPT, and other MCP-compatible tools can read pages, find blocks, and update content directly.

AI-Powered Editing

Describe what you want in plain language. The AI reads your page structure, locates the right blocks, and makes precise edits. No clicking through menus, no WYSIWYG wrestling.

"Change the pricing section to show three tiers instead of two, and update the CTA to say Get Started Free."

Simple Backend

Pure PHP. Flat-file storage. No database, no migrations, no ORM. Upload the files to any PHP host and it runs. The entire CMS is a handful of files you can read in an afternoon.

Under the Hood

Minimal moving parts. Maximum transparency. Everything you need, nothing you don't.

Architecture

1

Flat-file storage

Pages are HTML files on disk. No database, no serialization. What you see in the file is what gets served.

2

Block parsing

HTML comment markers define editable regions. The parser extracts, replaces, and writes blocks without touching surrounding markup.

3

Pure PHP

No framework. No Composer packages. A single-file entry point with a few helper includes. Runs on any PHP 8.0+ host.

4

Zero build step

No webpack, no node_modules, no compilation. Upload and run.

MCP Tools

list_pages // Get all managed pages
get_page // Read a page and its blocks
list_blocks // Enumerate blocks on a page
get_block // Read one block's content
update_block // Replace a block's HTML
create_block // Add a new block to a page
delete_block // Remove a block from a page
Read operations are safe. Write operations require auth.

Protocol: JSON-RPC 2.0 over stdio or HTTP, per the MCP specification.
